# 0020. Valid Parentheses

## Description

Given a string `s` containing just the characters `(`, `)`, `{`, `}`, `[` and `]`,
determine if the input string is valid.

A string is valid if:

- Open brackets are closed by the same type of brackets.
- Open brackets are closed in the correct order.
- Every closing bracket has a corresponding opening bracket.

---

## Example

### Example 1

**Input**

```text
s = "()"
```

**Output**

```text
true
```

---

### Example 2

**Input**

```text
s = "()[]{}"
```

**Output**

```text
true
```

---

### Example 3

**Input**

```text
s = "([)]"
```

**Output**

```text
false
```

---

# 💻 My Solution

## Solution 1 - Stack

```java
class Solution {
    public boolean isValid(String s) {

        Stack<Character> stack = new Stack<>();

        for (int i = 0; i < s.length(); i++) {

            char c = s.charAt(i);

            if (c == '(' || c == '[' || c == '{') {
                stack.push(c);
            } else {

                if (stack.isEmpty()) {
                    return false;
                }

                if (c == ')' && stack.peek() == '(') {
                    stack.pop();
                } else if (c == ']' && stack.peek() == '[') {
                    stack.pop();
                } else if (c == '}' && stack.peek() == '{') {
                    stack.pop();
                } else {
                    return false;
                }
            }
        }

        return stack.isEmpty();
    }
}
```

使用 **Stack** 來儲存尚未配對完成的左括號。
當遇到左括號時，將其放入 Stack；當遇到右括號時，先確認 Stack 是否為空，再檢查 Stack 最上層元素是否與目前的右括號相對應。如果配對成功則移除最上層元素，否則直接回傳 `false`。
最後如果 Stack 為空，代表所有括號皆成功配對；若 Stack 仍有元素，表示仍存在未配對的左括號，因此回傳 `false`。

# 📖 Note

## Stack

建立 Stack：

```java
Stack<Character> stack = new Stack<>();
```

Stack 是一種 **LIFO (Last In, First Out)** 的資料結構。

也就是：

> 最後放入的元素，會最先被取出。

---

### 常用方法

```java
stack.push(element);
```

放入元素。

```java
stack.peek();
```

查看最上層元素，不會移除。

```java
stack.pop();
```

移除最上層元素。

```java
stack.isEmpty();
```

判斷 Stack 是否為空。

---

## Why Stack?

以：

```text
({[]})
```

為例：

```text
(
({
({[
({      <- ] 配對成功
(       <- } 配對成功
空      <- ) 配對成功
```

每次都只能與**最近尚未配對完成**的左括號配對，因此非常符合 **Stack (LIFO)** 的特性。

若遇到：

```text
([)]
```

Stack 變化如下：

```text
(
([
```

當讀到 `)` 時，Stack 最上層為 `[`，無法配對，因此直接回傳 `false`。

---

## Time Complexity

```text
O(n)
```

每個字元最多只會被放入 (`push`) 與移除 (`pop`) 一次，因此只需遍歷字串一次。

---

## Space Complexity

```text
O(n)
```

最壞情況下，所有字元皆為左括號，需要全部存放於 Stack 中。
