# 0217. Contains Duplicate

## Description

Given an integer array `nums`, return `true` if any value appears at least twice in the array, and return `false` if every element is distinct.

---

## Example

### Example 1

**Input**

```text
nums = [1,2,3,1]
```

**Output**

```text
true
```

---

### Example 2

**Input**

```text
nums = [1,2,3,4]
```

**Output**

```text
false
```

---

### Example 3

**Input**

```text
nums = [1,1,1,3,3,4,3,2,4,2]
```

**Output**

```text
true
```

---

# 💻 My Solution

## Solution 1 - Brute Force

```java
class Solution {
    public boolean containsDuplicate(int[] nums) {

        for (int i = 0; i < nums.length; i++) {

            for (int j = i + 1; j < nums.length; j++) {

                if (nums[i] == nums[j]) {
                    return true;
                }

            }

        }

        return false;
    }
}
```

---

## Solution 2 - HashSet

```java
class Solution {
    public boolean containsDuplicate(int[] nums) {

        Set<Integer> seen = new HashSet<>();

        for (int i = 0; i < nums.length; i++) {

            if (seen.contains(nums[i])) {
                return true;
            }

            seen.add(nums[i]);
        }

        return false;
    }
}
```

先使用暴力 `for...for` 解法，雖然可以正確找出重複元素，但需要將每個元素與後面的所有元素逐一比較，因此時間複雜度為 **O(n²)**。

分析後可以發現，第二層 `for` 的真正目的其實只是一直確認：

> **目前這個數字以前有沒有出現過？**

因此改用 **HashSet**。

利用 `HashSet` 可以將已經看過的數字記錄起來，每次只需要使用 `contains()` 判斷是否已經存在。

如果已經存在，代表找到重複元素，可以直接回傳 `true`；如果不存在，就使用 `add()` 將目前數字加入 HashSet。

因此可以省略第二層 `for` 的搜尋，將時間複雜度從 **O(n²)** 優化為 **O(n)**。

# 📖 Note

## HashSet

建立 HashSet：

```java
Set<Integer> seen = new HashSet<>();
```

HashSet 只儲存元素本身，不會記錄 Index。

例如：

```text
5
2
8
```

HashSet：

```text
{5,2,8}
```

---

常用方法：

```java
seen.add(value)
```

新增元素。

```java
seen.contains(value)
```

判斷元素是否存在。

```java
seen.remove(value)
```

移除元素。

---

## HashSet vs HashMap

| HashSet  | HashMap                |
| -------- | ---------------------- |
| Value    | Key → Value            |
| 判斷元素是否存在 | 紀錄額外資訊（例如 Index、Count） |

HashSet：

```text
5
2
8
```

HashMap：

```text
5 -> index
2 -> index
8 -> index
```

---

## Learning Summary

* 暴力解法需要兩層 `for`，時間複雜度為 **O(n²)**。
* 第二層 `for` 的真正目的，是一直確認某個數字是否曾經出現過。
* HashSet 適合解決「是否存在」的問題。
* `contains()` 用來判斷元素是否存在。
* `add()` 用來將目前元素加入 HashSet。
* HashSet 不會自己記錄資料，必須透過 `add()` 將元素加入。
* HashSet 可以將 Contains Duplicate 從 **O(n²)** 優化為 **O(n)**。
* **HashSet 適合判斷是否存在；HashMap 適合除了存在之外，還需要記錄其他資訊（例如 Index、Count）。**
